안드로이드 9 이상의 환경에서 호스트 파일 복사에 실패됩니다

최신버전의 안드로이드는 /system 파티션을 읽기 모드만 사용하게 되어졌습니다.
만약 루팅을 Magisk로 하셨다면, 시스템 없는 호스트 모듈을 사용하도록 설정한 다음 재부팅하십시오.

파티션 공간 부족

환경설정에서 대상 호스트 파일을 /data/data/hosts (혹은 /data/hosts)로 변경한 후 AdAway를 다시 적용하십시오.

Chrome에서 여전히 광고가 나옵니다

Chrome 설정에서 라이트 모드(이전에는 데이터 세이버로 명칭 됨)가 비활성화 되어있는지 확인하십시오. Chrome은 이 기능을 통해 전용 DNS 솔루션을 사용하고 AdAway를 우회할 수 있습니다.

XYZ 앱에서는 광고를 차단하지 않습니다!

일부 호스트 이름은 호스트 소스에서 일부 호스트 이름이 누락될 수 있거나 혹은 응용 프로그램이 인터넷에 액세스하지 않고 광고를 제공하기 위해 이미지만 사용했을지도 모릅니다.

AdAway에서 DNS 요청(메뉴->Log DNS 요청)을 로깅하여 차단해야 하는 추가 호스트 이름을 확인할 수 있습니다.

로그의 항목을 길게 눌러 의심스러운 호스트 이름을 개인 블랙리스트에 추가하고 GitHub 문제 추적기에서 확인한 후 이 호스트 이름을 보고하십시오.

캐시된 광고

간혹 앱이 다운로드 이후 광고를 캐싱합니다. 이로 인해 일부 앱에 광고가 잔존하게 됩니다. 이 문제를 피하려면 안드로이드의 앱 목록에서 해당 앱의 캐시를 삭제를 시도해보십시오.

XYZ 앱은 더 이상 작동하지 않습니다!

일부 응용프로그램들은 광고를 제공해야 하는 호스트에 연결할 수 없을 때 AdAway에 의해 차단되거나 작동을 거부하는 호스트와 통신해야 합니다. 문제가 있는 알려진 응용 프로그램 목록을 확인하려면 https://github.com/AdAway/AdAway/wiki/ProblematicApps에서 참조하십시오. 아니면, 필요한 호스트 이름을 찾아 사용자 목록에 있는 화이트리스트에 추가한 후 AdAway 버그 트래커에 보고하십시오.

안드로이드 4.4 이상에서는 작동하지 않음

환경설정에서 대상 호스트 파일을 /data/data/hosts 에서 /data/hosts 혹은 /system/etc/hosts 으로 변경한 후 AdAway를 다시 적용하십시오.

웹 브라우저에서 뒤로 가기 버튼이 작동하지 않음

해결 방법으로 환경 설정에서 AdAway의 로컬 서버를 활성화 할 수 있습니다.
